# 0.1.2 — 2026-06-27

## Changed

- **`EVALS_DATA_DIR`** now defaults to `./evals-data` when unset or blank, so bare `npx` / `bunx` invocations start without any env setup ([#2](https://github.com/cyanheads/evals-mcp-server/issues/2)). A blank or whitespace-only value (the empty `.mcpb` Desktop directory field arrives as `""`) is treated as unset and resolves to the default rather than aborting startup. The variable is now marked optional in `server.json` and `manifest.json`.
- **`evals_get_record`** description dropped the meta-coaching sentence that named the verification-subagent caller and workflow step; it now states the action and the id-stable-across-submit behavior only ([#3](https://github.com/cyanheads/evals-mcp-server/issues/3)).

## Fixed

- **`mcq_choice_mismatch`** — `evals_create_draft` and `evals_revise_draft` now report `mcq_choice_mismatch` when an `mcq` grader's `correct` is not one of `choices[]`, matching `evals_run_check` and the documented error contract; it previously reported `task_type_constraint` ([#1](https://github.com/cyanheads/evals-mcp-server/issues/1)). `task_type_constraint` is now reserved for structural violations (`mcq` without `choices`, `free_response` without an `llm_rubric` grader).
